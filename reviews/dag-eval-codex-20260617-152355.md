## 1. Critique

The current DAG gets the author’s intention right: it is a threat inventory, not a DAG proof of DiD identification. The caption is especially right that parallel trends is a restriction on changes over time, not a back-door adjustment set in this compact static graph.

Specific issues:

- `Ideology -> Policy timing`, `Ideology -> Neighbour policy`, and `Ideology -> pi` are defensible. They capture the core selection-into-timing problem: policy adoption is partly a symptom of the same gender-ideology movement that changes usage.
- `Policy timing -> pi` is the correct target edge.
- `Neighbour policy -> f` is too compressed. Spillover may affect `pi` through speakers’ exposure, or `C` through shared wire/media material, or measurement/transcription conventions. A direct edge to `f` is fine as shorthand, but it hides the diagnostic.
- `pi -> f` is pedagogically useful but too coarse for this paper’s own estimand fact. Since the paper insists that observed frequency is latent usage plus composition/measurement, the graph should separate latent rate from coded corpus count through a measurement node.
- `Policy timing -> C -> f` labelled “mediator” is only conditionally right. It is right if policy changes the actual distribution of contexts, referents, topics, professions, or genres that are part of the causal process. It is not right if `C` is sampling-frame drift, outlet entry, archive coverage, author turnover, register mix, or topic shock not caused by policy. Then `C` is a bias source or adjustment target. It can also be a collider if `C` is affected both by policy and by topic/ideology/latent usage; conditioning or reweighting on it can open paths.
- Missing nodes matter: time, topic/news cycle, author/speaker turnover, genre/register mix, sampling frame, and transcription/coding convention. These can be collapsed into `C` for visual economy, but then the caption should not call `C` simply a mediator.
- The “threat inventory, not identification graph” framing is sound, but slightly overclaims. This compact static DAG cannot show parallel trends; a time-expanded graph or SWIG-style graph could show the relevant assumptions. If SWIGs are cited, the citation needs verification because no local key is present.

## 2. Alternative Structures

### A. Time-Expanded Two-Period DiD Graph

Nodes: `Group G`, `Ideology level U0`, `Ideology trend UΔ`, `Policy T1`, `Latent rate pi0`, `Untreated latent rate pi1(0)`, `Treated latent rate pi1(1)`, `Composition C0/C1`, `Measurement M0/M1`, `Observed f0/f1`.

Edges:  
`G -> T1`; `U0 -> T1`; `U0 -> pi0`; `U0 -> pi1(0)`; `UΔ -> T1`; `UΔ -> pi1(0)`; `T1 -> pi1(1)`; `T1 -> C1`; `pi0 -> f0`; `pi1(1) -> f1`; `C0 -> f0`; `C1 -> f1`; `M0 -> f0`; `M1 -> f1`.

Threat captured better: distinguishes fixed ideological level, which DiD can difference out, from ideology trend, which breaks parallel trends.

Cost: many nodes; risks turning the figure into a DiD tutorial for non-econometrician readers.

### B. Measurement-Expanded Corpus Graph

Nodes: `Ideology U`, `Policy timing T`, `Neighbour policy Tn`, `Latent usage pi`, `Composition C`, `Sampling frame S`, `Measurement/coding convention M`, `Coded count Y`, `Observed frequency f`.

Edges:  
`U -> T`; `U -> Tn`; `U -> pi`; `U -> C`; `T -> pi`; `T -> C`; `Tn -> pi`; `Tn -> C`; `S -> C`; `pi -> Y`; `C -> Y`; `M -> Y`; `Y -> f`.

Threat captured better: makes the key corpus claim visible: `f` is not `pi`; it is produced by sampling/composition plus coding/transcription. This directly serves Section 04 and the rung-3 transcription-convention warning.

Cost: less elegant; `f` becomes partly deterministic from `Y`, so the diagram is less like a pure causal DAG.

### C. Selection-Into-Treatment-Timing Graph

Nodes: `Ideology wave Wgt`, `Institutional threshold Hg`, `Adoption date Tg`, `Untreated rate trend pi0_gt`, `Observed latent rate pi_gt`, `Composition Cgt`, `Measurement Mgt`, `Observed fgt`.

Edges:  
`Wgt -> Tg`; `Hg -> Tg`; `Wgt -> pi0_gt`; `Wgt -> Cgt`; `Tg -> pi_gt`; `pi0_gt -> pi_gt`; `Cgt -> fgt`; `Mgt -> fgt`; `pi_gt -> fgt`.

Threat captured better: the paper’s irreducible false positive. A polity adopts when its own usage trend is already turning, so an event-time jump can appear even with zero policy effect.

Cost: more abstract; requires readers to understand treatment timing as a hazard process rather than a simple policy node.

## 3. Recommendation

Do not keep the current DAG exactly as is. Keep the threat-inventory framing, but switch the main figure toward Alternative B, with one feature from Alternative C: make the ideology node explicitly a time-varying ideology wave that drives both adoption timing and latent usage. That best serves a non-econometrician corpus-linguistics audience because it keeps the lesson concrete: the corpus frequency is made, not revealed. A full time-expanded DiD graph is methodologically cleaner, but it would overload the main figure and pull the paper toward a DiD tutorial. The current graph is close; its main flaw is not the threat-inventory stance, but the over-simple treatment of `C` as always a mediator and `pi -> f` as a single measurement link.
